# interview_plugins
## 插件化执行系统（面试项目）

本项目基于 Go 语言，实现了一套**轻量、高解耦、高可扩展**的插件化执行系统。

主程序完全不感知具体业务逻辑，仅通过统一插件接口，完成插件加载、调度、全生命周期管理，支持业务能力快速迭代扩展，无需频繁修改主框架核心代码。

## 2. 分层架构设计

*   **接口抽象层（plugin 包）**
    定义全局统一 `Plugin` 接口与插件标准契约，所有业务插件必须严格遵循该接口规范。

*   **全局注册中心层**
    统一收纳、管理所有自动注册的插件实例，实现插件集中托管与批量获取。

*   **核心管理层（manager 包）**
    提供插件加载/卸载、启用/禁用、执行调度、异常隔离、元信息查询等全套核心管理能力。

*   **业务插件层（plugins 目录）**
    独立开发、独立维护的具体业务插件，与主程序、管理器完全解耦。

*   **入口调度层（main.go）**
    负责系统启动初始化、插件批量加载、功能串联与流程演示。

---

## 3. 核心设计思路

1. **面向接口，遵循开闭原则**
    主程序与管理器全程仅依赖顶层抽象接口，不耦合任何具体插件实现。新增、修改、下线业务，仅需改动插件文件。

2. **插件自动发现、自动注册**
    插件通过 `init()` 函数 + 匿名导入机制，启动时自动注册，无需手动硬编码，真正做到即插即用。

3. **插件全生命周期状态管理**
    维护插件独立状态：`启用` / `禁用` / `异常`，运行时可随时开关，灵活管控可用性。

4. **强故障隔离机制**
    捕获插件运行 panic 与异常，单个插件崩溃**绝不影响主程序与其他插件**，自动标记异常状态，阻断故障扩散。

5. **架构极简、预留极强扩展能力**
    架构简洁干净，原生支持快速扩展热重载、超时控制、依赖管理、跨语言插件等高阶能力。

## 5. 进阶扩展设计方案

### 5.1 跨语言插件支持
*   定义全局统一 JSON 输入输出数据协议。
*   非 Go 语言插件，通过 HTTP / gRPC / 本地子进程接入调用。
*   主系统不关心插件语言，仅校验标准数据格式。
*   现有架构无需改动，仅新增一层通用调用适配层即可。

### 5.2 插件执行超时控制
*   使用 `context.WithTimeout` 为插件执行绑定独立超时时间。
*   插件运行超时后，主动终止执行、释放资源。
*   超时插件自动标记异常，后续不再调度。
*   全程不阻塞、不影响主程序流程。
