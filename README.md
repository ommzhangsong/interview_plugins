# interview_plugins
面试插件系统设计
插件介绍：本项目基于 Go 语言实现了一套轻量、高解耦、可扩展的插件化执行系统。主程序完全不感知具体业务逻辑，仅通过统一插件接口加载、调度、管理业务插件，满足业务能力动态扩展、无需频繁修改主程序的工程诉求。
1.项目目录结构：
awesomeProject/
├── go.mod              # Go模块依赖管理
├── main.go             # 程序入口、功能演示
├── plugin/
│   ├── plugin.go       # 统一插件接口定义、状态常量、基础实体
│   └── registry.go     # 全局插件自动注册中心
├── manager/
│   └── manager.go      # 插件核心管理器
└── plugins/
    ├── order.go        # 订单支付业务插件
    └── refund.go      # 退款处理业务插件

2.分层架构
接口抽象层（plugin 包）：定义全局统一Plugin接口与插件标准契约，所有插件必须遵循该规范。
注册中心层：统一收纳所有自动注册的插件实例，实现插件的集中托管。
核心管理层（manager 包）：提供插件加载卸载、启用禁用、执行调度、异常隔离、元信息查询等全部核心能力。
业务插件层（plugins 目录）：独立开发、独立维护的具体业务插件，与主程序完全解耦。
入口调度层（main.go）：系统启动、插件初始化、功能演示与流程调度。

3.插件系统核心设计思路
面向接口、开闭原则
主程序、管理器全程仅依赖顶层Plugin抽象接口，不依赖任何具体业务插件实现；新增、修改、下线业务逻辑，仅新增 / 改动插件文件，无需改动主框架代码。
自动发现与自动注册
插件通过init()方法 + 匿名导入机制，程序启动时自动完成注册，无需手动硬编码注册，真正做到插件即插即用。
运行状态全生命周期管理
为每个插件维护独立状态：启用/禁用/异常，支持运行中随时开关插件，灵活管控插件可用性。
强故障隔离设计
插件运行时捕获全部 panic 与运行异常，单个插件崩溃、业务报错，绝对不会导致主程序与其他插件崩溃，同时自动标记插件为异常状态，避免故障扩散。
极简可扩展
架构天然支持后续横向扩展热重载、超时控制、依赖管理等高阶能力，适配长期业务迭代。

5.进阶设计：
5.1 跨语言插件支持
实现思路
定义统一 JSON 输入输出协议
非 Go 插件通过 HTTP /gRPC/ 子进程 调用
主系统不关心插件语言，只认数据格式
插件化架构完全不变，只需增加一层调用适配
5.2 插件执行超时控制
实现思路
使用 context.WithTimeout 为插件执行设置最大执行时间（如 3 秒）
超时自动终止执行，不再等待插件返回
超时后标记插件为异常状态，防止后续继续调用
主程序不受影响，直接返回超时错误
